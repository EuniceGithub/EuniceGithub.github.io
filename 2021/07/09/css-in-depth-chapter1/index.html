Cascading Rule (冲突筛选规则) 的具体内容

如第一张图所示，有冲突的 CSS 语句最多要经过三个筛选层后才能确定最终选用的值。顺带提一句，该最终值被称为 cascaded value。

  • 第一层，筛选来源 (origin)。
  • 第二层，筛选针对性 (specificity)。
  • 第三层，语句的出现顺序。

在每一各筛选层内部，“优先级”高的语句将胜出。如果只有一条语句胜出，结果当场就确定了，如果有多条语句是相同的最高优先级，则将它们挑出，进入下一个筛选层，非最高优先级的那些语句抛弃掉不再考虑。

判断优先级的规则，每一层是各不相同的，毕竟每一层筛选所考察的角度不同。接下来说每一层的优先级是如何判断的。

【第一层，来源】

有如下三个优先级:

  1. 带有 !important 修饰的, 优先级最高。
  2. 用户在 html/css 源码中写的, 优先级次高. 在 html 中写的，不论 inline 与否。
  3. 浏览器(user agent)默认施加的, 优先级最低. 比如浏览器会默认给链接文本显示为蓝色加下划线。

这里特别要提一下·!important这个“来源”。虽然形式上它不像是一种“来源”，但实质上是。比如，在 css 中有一个 color 语句被标记了 !important，在 .html 中有个 元素的 inline style 中的 color 也被标记了 !important ，虽然两处 !important 分散在不同性质的文件中，我们还是得认为它们是同一个 origin，具有相同的 origin 优先级。

【第二层，针对性】

这个筛选层的规则稍微复杂一点。 总体而言，针对性越强的，优先级越高。

针对性是什么意思呢? 沿用前头的例子，

  • h1 { color: red; } 针对的是所有的 h1 元素。 h1 这种形式被称为 tag selector(或 type selector).（标签选择器）
  • .title { color: green; } 针对的是 class=title 的所有元素。 .title 这种形式被称为 class selector.（类选择器）
  • #page-title { color: blue; } 针对的是 id=page-title 的这一个元素。 #page-title 这种形式被称为 ID selector.(ID选择器）
  • HTML 中写
    <p style="color:orange; font-weight:bold;">some text</p>
    的话，这种写法称为 inline style， 针对的就是当前这个 p 元素。

首先, inline style 的优先级高, 因为它的针对性最强, 最无可异议.

接下来的规则是 ID selector 高于 class selector, 高于 tag selector.

更具体地说，如果一个规则中有多个selector（选择器）, 那么规则如下:

  • 具有最多 ID selector 的语句胜出（实质是该 CSS 语句对应的属性值胜出）。

  • ID selector 数目相同的话，具有最多 class selector 的语句胜出。

  • class selector 数目相同的话，具有最多 tag selector 的语句胜出。

    ​ 说明　伪类选择器（如:hover）和属性选择器（如[type="input"]）与一个类选择器的优先级相同。通用选择器（*）和组合器（>、+、~）对优先级没有影响。

    ​ *组合器：

    ​ CSS 中有四种不同的组合器：

    • 后代选择器 (空格)
    • 子选择器 (>)
    • 相邻兄弟选择器 (+)
    • 通用兄弟选择器 (~)
优先标记级：

人们为了表达方便, 将一条 css 语句的针对性用一个三元组来表示. 比如，“1,2,2” 表示一个 ID selector、两个 class selector、两个 tag selector. 摆在前头的数字权重比较大, 因此, 比较两个 css 语句的针对性的方法是, 依次比较它们三元组中的数字, 权重大的胜出.

链接样式和源码顺序

​ 给链接加样式要按照一定的顺序书写选择器。这是因为源码顺序影响了层叠。

​ 记忆口诀是“LoVe/HAte”（“爱/恨”），即link（链接）、visited（访问）、hover（悬停）、active（激活）

处理层叠时有两条通用的经验法则。

(1) 在选择器中不要使用ID。就算只用一个ID，也会大幅提升优先级。当需要覆盖这个选择器时，通常找不到另一个有意义的ID，于是就会复制原来的选择器，然后加上另一个类，让它区别于想要覆盖的选择器。

(2) **不要使用!important**。它比ID更难覆盖，一旦用了它，想要覆盖原先的声明，就需要再加上一个!important，而且依然要处理优先级的问题。

这两条规则是很好的建议，但不必固守它们。

继承

如果一个元素的某个属性没有层叠值，则可能会继承某个祖先元素的值。

不是所有的属性都能被继承。默认情况下，只有特定的一些属性能被继承。

它们主要是

跟文本相关的属性：color、font、font-family、font-size、font-weight、font-variant、font-style、line-height、letter-spacing、text-align、text-indent、text-transform、white-space以及word-spacing。

列表属性：list-style、list-style-type、list-style-position以及list-style-image。

表格的边框属性border-collapse和border-spacing

inherit关键字

用继承代替一个层叠值可以用inherit关键字。

它可以覆盖另一个值，这样该元素就会继承其父元素的值。

initial关键字

initial关键字可以撤销作用于某个元素的样式，将其重置为默认值。

这么做的好处是不需要思考太多。如果想删除一个元素的边框，设置border: initial即可。如果想让一个元素恢复到默认宽度，设置width: initial即可。

简写属性

简写属性是用于同时给多个属性赋值的属性。

  • font是一个简写属性，它指定了font-style、font-weight、font-size、font-height以及font-family。
  • background是多个背景属性的简写属性：background-color、background-image、background-size、background-repeat、background-position、background-origin、background-chip以及background-attachment。
  • border是border-width、border-style以及border-color的简写属性，而这几个属性也都是简写属性：
  • border-width、border-style、border-color是上、右、下、左四个边框宽度、样式、颜色的简写属性。（注意顺序）

注：大多数简写属性可以省略一些值，只指定我们关注的值。省略的值会被设置为初始值，默默覆盖在其他地方定义的样式。

简写属性值的顺序

1.上、右、下、左（按顺时针方向，从上边开始）

用于为元素的四条边分别指定值的边框属性时，如margin（外边距）,padding（内边距）。记忆口诀是 TRouBLe ：top（上）、right（右）、bottom（下）、left（左）。

如果声明结束时属性值还有没指定，没有指定的一边会取其对边的值。只指定一个值时，所有边都取这一个值。

对于较小的元素，左右的内边距最好大于上下内边距。这种样式很适合网页的按钮或者导航链接

2.水平、垂直

用于属性支持最多指定两个值时，这些属性包括background-position、box-shadow、text-shadow

总结
  • 控制选择器的优先级。
  • 不要混淆层叠和继承。
  • 某些属性会被继承，包括文本、列表、表格边框相关的属性。
  • 不要混淆initial和auto值。
  • 简写属性要注意TRouBLe的顺序，避免踩坑。
